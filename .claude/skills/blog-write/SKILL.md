## 개요

이 스킬은 LLM 대화 기록(prompt)과 요약(summary)을 기반으로 블로그 글을 작성한다.
팀 방식으로 2~3명의 작가 에이전트가 각자 다른 스타일로 초안을 쓰고,
편집장 에이전트가 심사하여 최종본을 완성한다.

## Step 1: 소재 파일 찾기 및 디렉토리 정리

### 사용자 준비
사용자는 `drafts/` 폴더 바로 아래에 다음 파일을 놓아둔다:
- `prompts.json` (필수) — AI Studio 대화 내보내기 파일
- `summary.md` (옵션) — 대화 요약. 없으면 prompt.md에서 자동 생성.

### 파일 탐색
`drafts/` 에서 소재 파일을 찾는다. 탐색 순서:

1. **drafts/ 루트에 prompts.json 또는 prompt.md가 있는 경우** (새 글 작성)
   - prompt.md를 읽어 주제 키워드를 추출
   - `drafts/YYYY-MM-DD-{주제키워드}/` 디렉토리를 생성 (오늘 날짜 사용)
   - 소재 파일(prompt.md, prompts.json, summary.md)을 새 디렉토리로 이동
   - summary.md가 없으면 prompt.md에서 핵심 내용을 요약하여 summary.md를 자동 생성
   - 생성된 디렉토리와 파일을 사용자에게 보고

2. **drafts/ 루트에 파일이 없는 경우** (기존 글 이어쓰기)
   - drafts/ 하위 폴더 중 가장 최근 수정된 폴더를 사용
   - 해당 폴더에 prompt.md가 있는지 확인

3. **아무것도 없는 경우**
   - 사용자에게 "drafts/ 폴더에 prompts.json을 넣어주세요"라고 안내하고 중단

### prompt.md 변환
**모든 단계에서 prompt.md를 대화 소재의 기본 파일로 사용한다.**
prompt.md가 없고 prompts.json이 있으면, 추출 스크립트를 실행하여 생성한다:
```bash
python scripts/extract-prompts.py <prompts.json 경로>
```
변환 완료 후 prompts.json은 삭제한다. prompt.md를 기준으로 진행한다.

### 폴더명 규칙
- 형식: `YYYY-MM-DD-{주제키워드}` (예: `2026-03-27-물고기-지느러미`)
- 주제 키워드는 prompts.json 또는 summary.md에서 핵심 단어 2~3개를 추출하여 하이픈으로 연결
- 한글 사용 가능

## Step 2: 소재 분석 및 스타일 추천

summary.md를 읽고 주제를 파악한다. 그 후 styles/ 폴더의 스타일 파일들을 읽고,
주제에 가장 적합한 2~3개 스타일을 선택한다.

**스타일 매칭 기준:**
- 철학적 사고, 인간 본성 → bertrand-russell, kurt-vonnegut, mark-twain
- 뇌과학, 자연과학 → bill-bryson, bertrand-russell, douglas-adams
- AI, 기술 → douglas-adams, kurt-vonnegut, malcolm-gladwell
- 사회 현상, 트렌드 → malcolm-gladwell, mark-twain, bertrand-russell
- 데이터, 심리학 → malcolm-gladwell, bill-bryson

위는 참고 기준이며, 소재의 톤과 깊이에 따라 유연하게 판단한다.

선택한 스타일을 사용자에게 보고하고, 확인 없이 바로 Step 3로 진행한다:
```
📝 주제: [주제 요약] | ✍️ 스타일: [스타일1], [스타일2], [스타일3]
```

사용자가 특정 스타일을 지정한 경우 해당 스타일을 반드시 포함한다.

## Step 3: 팀 작성 (병렬)

스타일 선택 후 바로 2~3개의 **writer 에이전트**를 병렬로 실행한다.

각 writer 에이전트에게 전달할 정보:
1. summary.md 내용
2. prompt.md에서 핵심 대화 (너무 크면 요약)
3. 해당 스타일 파일 (styles/{style-name}.md) 내용
4. `reviews/feedback-log.md` 내용 (파일이 있는 경우). 해당 스타일의 과거 피드백을 반드시 반영하도록 지시
5. 아래 작성 지침

**작성 지침:**
- 한글로 작성
- 2000~4000자 분량
- 블로그 독자 대상 (전문가가 아닌 지적 호기심이 있는 일반 독자)
- 스타일 파일의 특징, 기법, 피해야 할 것을 충실히 따를 것
- 대화에서 나온 핵심 인사이트와 논리 흐름을 살릴 것
- 제목(H1)과 소제목(H2) 포함


각 writer는 결과를 다음 형식으로 반환한다:
```
## 스타일: {스타일명}
## 제목: {제목}

{본문}
```

**각 writer의 결과물을 즉시 파일로 저장한다:**
- 저장 경로: `drafts/{폴더명}/draft-{스타일명}.md` (예: `draft-bertrand-russell.md`)
- 스타일명은 styles/ 폴더의 파일명과 동일하게 영문 소문자-하이픈 형식

writer 에이전트 정의: `agents/writer.md` 참조

## Step 4: 편집장 심사

3개의 초안이 모이면 **editor 에이전트**를 실행한다.

editor 에이전트의 역할:
1. 각 초안의 강점과 약점을 분석
2. 다음 중 하나를 결정:
   - **단일 선택**: 가장 우수한 초안을 선택하고 소폭 수정
   - **하이브리드**: 최고의 초안을 기반으로 다른 초안의 좋은 부분을 차용 (단, 톤의 일관성 유지)
3. 최종본 작성

editor는 심사 결과를 다음 형식으로 보고한다:
```
## 심사 결과

### 초안 비교
| 스타일 | 강점 | 약점 | 점수(10) |
|--------|------|------|---------|
| ... | ... | ... | ... |

### 결정: [단일 선택 / 하이브리드]
### 이유: ...

## 최종본
{최종 한글 본문}
```

**편집장의 심사 결과와 최종본을 파일로 저장한다:**
- 심사 결과: `drafts/{폴더명}/review.md`
- 최종 초안: `drafts/{폴더명}/draft-final.md`

**피드백 로그 업데이트:**
편집장 심사 완료 후 (그리고 사용자 피드백 반영 후), `reviews/feedback-log.md`에 이번 회차의 핵심 교훈을 추가한다:
- 날짜와 주제
- 스타일별 강점/약점 요약 (2~3문장씩)
- 사용자가 추가로 준 피드백
- 일반 교훈 (다음 글에 적용할 점)
- 로그가 너무 길어지면 오래된 항목의 세부사항을 압축하되, 핵심 교훈은 유지

editor 에이전트 정의: `agents/editor.md` 참조

## Step 5: 사용자 확인 및 수정

최종본을 사용자에게 보여준다. 수정 요청이 있으면 반영한다.

## Step 6: 메타데이터 생성

최종본을 바탕으로 다음을 생성한다:

### 제목 & 부제
- 제목: 호기심을 자극하는 짧은 문장 (15자 이내 권장)
- 부제: 핵심 내용을 요약하는 한 문장
- 3개 후보를 제시하고 사용자가 선택

### 태그
- 5~8개 태그 제안
- Substack에서 검색 가능한 키워드 포함
- 한글 + 영문 태그 모두 제안

### Substack Note
- 블로그 글을 홍보하는 짧은 Note 텍스트 (한글/영문 각 1개)
- 핵심 인사이트를 담은 훅(hook)으로 시작하여 글로 유도
- 280자 이내, 캐주얼한 톤
- 글 링크 자리를 `[링크]`로 표시

### 이미지 프롬프트
- Gemini (Nano Banana) 용 이미지 생성 프롬프트
- 글의 핵심 컨셉을 시각적으로 표현
- 2~3개 후보 제시
- 형식: 영문 프롬프트 + 스타일 지시 (예: "digital art", "watercolor", "minimalist illustration")

## Step 7: 영문 번역

확정된 한글 본문을 영어로 번역한다.

**번역 지침:**
- 직역이 아닌 자연스러운 영문 에세이로 재작성
- 원문의 유머와 톤을 영어 화자에게 맞게 조정
- 한국 문화 고유의 표현은 적절한 영어 대응어로 변환
- 제목, 부제, 태그도 영문 버전 생성

## Step 8: published에 저장 및 완료

확정된 모든 파일을 `published/YYYY-MM-DD-{주제키워드}/` 폴더에 저장한다.
저장 파일:
- `post-ko.md` — 한글 최종본
- `post-en.md` — 영문 번역본
- `image-prompt.md` — 이미지 생성 프롬프트
- `meta.md` — 제목, 부제, 태그, Substack Note (한/영)

완료 후 요약을 출력한다:

```
블로그 작성 완료!

폴더: published/{folder-name}/
  - post-ko.md
  - post-en.md
  - image-prompt.md
  - meta.md

제목(KO): ...
제목(EN): ...
태그: ...
사용 스타일: ...

다음 단계: Substack에 수동 게시
```

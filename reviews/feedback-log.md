# 편집장 피드백 로그

작가 에이전트가 글을 쓸 때 참조하는 누적 피드백.
편집장이 심사할 때마다 핵심 교훈이 여기에 추가된다.

---

## 2026-03-27 — 지능-진화-문어

### 스타일별 피드백

**빌 브라이슨** (선택됨, 수정 후 최종 채택)
- 자기비하 유머가 "솔직히 뿌듯했다" 하나뿐이었음 → 3~4회 분산 배치 필요
- 인물 일화/실화 에피소드가 전혀 없었음 → 최소 1~2개 삽입 필요
- 결론이 칼럼니스트풍("인공지능 설계에서 다시 마주하고 있다")이었음 → 브라이슨 톤으로 마무리해야 함
- **잘한 점**: 점층적 전개(다이빙→의문→발견→더 큰 발견), "처음 이 사실을 알았을 때 나는 잠시 멍하니 있었다" 같은 경이감 표현

**버트런드 러셀** (초기 편집장 선택, 사용자 거부)
- 문장 단위 밀도는 높았으나, 매 문단마다 비트는 패턴이 "산만하게" 느껴짐
- 사용자 피드백: 전개가 따라가기 어려움. 선형 서사가 더 편함
- **교훈**: 러셀 스타일 사용 시 논리의 뼈대를 더 단순하게 유지할 것

**더글러스 애덤스**
- 후반부로 갈수록 톤이 진지해지며 애덤스다움이 사라짐
- "이해하지 못하는 쪽이 더 행복한지도 모른다" — 감상적 마무리로 스타일 위반
- **교훈**: 끝까지 부조리 톤 유지. 마지막 문장이 가장 엉뚱해야 함

### 일반 교훈
- 사용자는 **가독성과 자연스러운 전개**를 문장의 밀도보다 우선시함
- 실화 에피소드(인키 탈출, 시애틀 수족관 야간 범행)가 글의 설득력을 크게 높임
- 자기비하 유머의 핵심 패턴: **대단한 사실 직후 → 자기의 무능/당혹 고백**

## 2026-03-29 — 도덕적원-확장

### 방향 수정 (v1 → v2)
- v1은 "도덕적 원이 확장되고 있다 = 좋은 일"이라는 낙관적 결론을 담고 있었음
- 사용자 피드백: 확장에 대해 아무 의견이 없다. 관찰에서 멈춰라
- v2에서 변증법적 해결, 낙관적 결론, 해결책 제시 모두 삭제
- **summary.md를 "글의 방향 길잡이"로 재정의** — 다루는 내용뿐 아니라 "다루지 않는 내용"도 명시

### 스타일별 피드백 (v2)

**빌 브라이슨** (하이브리드 기반)
- 도입부와 개인 서사가 자연스러움
- 자기비하 유머와 점층 전개가 효과적
- 약점: 중반부 이론 밀집 시 강의 톤

**마크 트웨인**
- "관광객의 원은 넓지만 가볍다. 마부의 원은 좁지만 무겁다" — 최종본에 차용
- "누구의 천국인가" 시작이 날카로움
- 방향 준수가 가장 정확

**버트런드 러셀**
- "친환경이라 부르는 것은 용어의 사용법에 대한 흥미로운 사례" — 최종본에 차용
- "말이 나보다 나은 처지" 마무리 — 최종본에 차용
- 이전보다 선형 서사가 개선되었으나 여전히 밀도가 높음

### 사용자 피드백 (누적)
- 말 상태 묘사 과장 금지 → 현실에 맞게 서술
- 서양/동양 이분법 사용 금지
- 학술 용어로 같은 말 반복 금지 (죄수의 딜레마 등)
- 사실이 아닌 체험 꾸며내기 금지
- 팩트체크 필수
- 글이 가치 판단이나 낙관적 결론을 내리지 않도록 주의
- **사용자는 특정 스타일을 고정 선호하지 않음** — 편집장이 "사용자가 X 스타일 선호"라고 가정하지 말 것

### 일반 교훈
- summary.md가 글의 범위와 톤을 결정하는 핵심 도구. "다루지 않는 내용"을 명시하면 작가 에이전트의 범위 이탈을 방지
- 하이브리드 방식 유효: 한 스타일 기반 + 다른 스타일의 날카로운 한 줄 차용
- 이론/개념이 연속되면 강의 톤 → 섹션당 개념 1개
- 체험 기반 글에서는 사실 확인 후 서술

## 2026-03-31 — AI-에이전트-관심경제

### 워크플로우 변경
- prompt.md를 summary.md에 통합 후 삭제하는 방식으로 변경. summary.md의 "글에 활용할 소재" 섹션에 통계/사례/비유/핵심 문장을 풍부하게 담아 작가가 prompt.md 없이도 글을 쓸 수 있게 함
- 결과: 3명의 작가 모두 소재를 잘 활용함. prompt.md 없이도 충분

### 스타일별 피드백

**말콤 글래드웰** (첫 사용)
- 구조가 가장 탄탄: 슈퍼볼→슈퍼볼 원형 구조, 반직관 전개 잘 됨
- 약점: 분야 횡단 사례 부족 (주제 안에서만 움직임), 개인적 목소리 거의 없어 분석 기사처럼 읽힘
- 마지막 문단이 다소 장중한 결론 느낌 — "관찰에서 멈추는" 톤에서 벗어남
- **교훈**: 글래드웰 스타일은 반드시 주제 외 분야의 에피소드를 1~2개 병치해야 함. 개인 서사 추가 필요

**빌 브라이슨** (하이브리드 기반으로 최종 채택)
- 가독성 최고, 1인칭 관찰자 시점이 원본 톤과 잘 맞음
- "식당 비유", 솔직한 자기비하가 효과적
- 약점: 탈선이 부족 (너무 직선적), 유머가 미소 수준이지 웃음까지 안 감
- **교훈**: 브라이슨은 본론 중간에 관련 있지만 의외인 일화로 탈선했다가 돌아오는 게 핵심. 다음에는 의도적 탈선 1~2회 삽입

**더글러스 애덤스**
- 유머 최고: "은하 여행자 안내서" 비유, "밥도 안 먹고, 화장실도 안 가고" 리스트, 마지막 "트레킹화" 마무리 탁월
- 이전 피드백(끝까지 부조리 톤 유지) 잘 반영됨 — 감상적 마무리 없음
- 약점: 구조 느슨, 봇 트래픽 데이터 반복
- **교훈**: 애덤스 스타일의 부조리 유머는 하이브리드 차용 시 매우 효과적. 마무리 기법이 특히 좋음

### 일반 교훈
- summary.md에 소재를 풍부하게 담으면 prompt.md 없이도 작가가 충분히 활용 가능
- 하이브리드 3중 차용(Bryson 베이스 + Gladwell 에피소드 + Adams 마무리)이 잘 작동함
- 팩트체크를 별도 단계로 진행 — 13개 항목 중 11개 확인, 2개 부분적 맞음, 틀린 것 없음
- 사용자는 팩트 정확성을 매우 중시함

## 2026-04-12 — 이야기-황금알

### 스타일 선택
새로 추가된 스타일 3개 사용: Joan Didion, David Sedaris, Nora Ephron (Vonnegut 삭제 후 교체)

### 스타일별 피드백

**David Sedaris** (하이브리드 기반, 8/10)
- 체험 밀도가 가장 높음, 자기비하 유머 자연스러움
- 마지막 문단("깔끔한 결론을 경계하도록 훈련된 편집장")이 최고 엔딩
- 약점: 갓셜/하라리 연속 인용이 강의 톤

**Joan Didion** (7/10)
- 건조한 톤 일관적, "플라스틱 꽃" 비유 효과적
- 약점: 전개가 교과서적으로 정돈되어 오히려 강의 톤

**Nora Ephron** (6.5/10)
- 가장 읽기 편하지만 가장 특색 없음
- "시들지 않는다는 장점이 있지만, 그게 정확히 단점이기도 하다" — 날카로운 한 줄 차용

### 핵심 과정: Codex 비교 경쟁
- Codex(OpenAI)에게도 같은 소재로 초안을 받아 비교
- Codex가 유머 밀도에서 압도적으로 우세 — "공동묘지", "김밥천국 자동화", "배송받는다", "젖어본 적은 없다" 등
- 우리 글이 체험 구체성(러셀/브라이슨 에피소드)과 사례 정확성(수치)에서 우세
- **최종 전략**: 우리 글을 베이스로 Codex의 유머를 이식 → 가장 좋은 결과
- Codex의 "미묘하게 부끄러운 흥분" 표현을 결론부에 차용

### 사용자 에피소드 (직접 제공)
- AI가 처음 쓴 러셀풍 글을 읽고 "도파민이 터지는" 경험 — 프로젝트의 원동력
- 브라이슨이라는 작가를 AI를 통해 처음 알게 됨
- "작가가 이런 기분일까" — 글이 만들어지는 과정 자체의 즐거움
- 이 체험들이 초안에 없어서 "심심하다"는 피드백 → 추가 후 확연히 개선

### 팩트체크
- 5개 항목 중 4개 확인, 1개 수정
- "프로 작가의 45~60%가 AI 활용" → Authors Guild 조사 기준 23%. "상당수의 작가가"로 수정

### 일반 교훈
- **유머가 가장 큰 차이를 만든다** — 같은 구조, 같은 소재라도 유머 밀도가 글의 생동감을 결정
- 사용자의 실제 에피소드가 글의 핵심. "체험 꾸며내기 금지"이므로 사용자에게 적극적으로 에피소드를 요청할 것
- 복수 AI 비교(Codex vs Claude)가 효과적인 퇴고 방법이 될 수 있음
- 리뷰 피드백(논지 축, 사례 선행, 비유 연결, 제목 예고)이 구조 개선에 유효했음. 단, 열린 결말에 실천 방향 추가는 비동의 — 고유 톤("관찰에서 멈추기") 우선
